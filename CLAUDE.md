# Pension-tracer — Claude 작업 규칙

## ⚠️ 코드 작업 금지 규칙 (최우선)

**사용자가 "코드작성"이라고 말하기 전까지 절대 코드를 작성하거나 수정하지 않는다.**

- 분석, 계획, 설명만 하고 대기
- "코드작성" 메시지를 받은 후에만 실제 파일 수정/생성 진행
- **각 새로운 작업마다** 반드시 "코드작성" 승인을 받아야 한다 (세션 내 1회 승인 ≠ 전체 세션 허용)
- 분석 요청 ("점검해", "확인해", "살펴봐" 등)은 절대 코드 수정 허가가 아님
- 모든 코드 작업이 완료되면 반드시 **"코드작업완료"** 라고 알린다
- 이 규칙은 다른 모든 규칙보다 우선한다

## PLAN.md — 항상 먼저 읽고, 항상 업데이트 (필수)

**대화 시작 시 또는 작업 전 반드시 `PLAN.md`를 읽어 현재 상태를 파악한다.**
**코드를 수정할 때마다 반드시 `PLAN.md`를 업데이트한다.**

> PLAN.md는 이 프로젝트의 유일한 진실 공급원(single source of truth)이다.

| 상황 | 해야 할 일 |
|------|-----------|
| 대화/작업 시작 | PLAN.md 읽기 — 현재 상태 파악 |
| 작업 완료 | 완료 테이블에 행 추가 |
| 새 작업 발견 | 남은 작업 목록에 추가 |
